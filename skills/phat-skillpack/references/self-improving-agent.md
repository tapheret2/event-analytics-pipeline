# self-improving-agent — learning loop (notes)

Maintain a lightweight learning log so repeated failures become durable fixes.

## What to log
- Command/tool failures → `.learnings/ERRORS.md`
- User corrections / knowledge gaps / best practices → `.learnings/LEARNINGS.md`
- Missing features users request → `.learnings/FEATURE_REQUESTS.md`

## Promotion targets (OpenClaw workspace)
- Behavioral patterns → `SOUL.md`
- Workflow improvements → `AGENTS.md`
- Tool gotchas → `TOOLS.md`
- Long-term user context → `MEMORY.md` (main session)

## Entry IDs
`TYPE-YYYYMMDD-XXX` where TYPE is LRN/ERR/FEAT.
